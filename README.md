## What is OwnLLM?

OwnLLM is an enhanced fork of [AnythingLLM](https://github.com/Mintplex-Labs/anything-llm) that transforms a general-purpose AI chat application into a **business document and proposal generation platform**.

Chat with your documents, generate AI-powered proposals with real pricing, edit in a rich text editor, and export branded PDFs — all self-hosted with full privacy.

## Key Features

### From AnythingLLM (Base)
- Chat with documents (PDF, DOCX, TXT, and more)
- RAG (Retrieval-Augmented Generation) with vector embeddings
- Support for **25+ LLM providers** (OpenAI, Claude, Gemini, Ollama, DeepSeek, Groq, Mistral, etc.)
- Support for **10+ vector databases** (LanceDB, Pinecone, Chroma, Qdrant, Milvus, PGVector, etc.)
- Multi-user with permissions
- AI Agents with custom tools and web browsing
- Embeddable chat widget
- Full REST API
- MCP compatibility

### OwnLLM Additions

| Feature | Description |
|---------|-------------|
| **Thread Notes** | Rich text editor (BlockNote/Yoopta) alongside every chat thread |
| **Branded PDF Export** | Export notes to professionally styled PDFs with custom logos, colors, and templates |
| **Products Manager** | Define products/services per workspace with pricing for AI-generated proposals |
| **Rate Card Manager** | Define roles and hourly rates for time & materials proposals |
| **Smart Plugins** | Per-workspace plugins with custom schemas |
| **Code Artifacts** | Save and manage code artifacts generated by AI |
| **Multi-Tenant SaaS** | Organizations with complete data isolation, billing, and seat limits |
| **Email Verification** | Full signup/verification flow |

### How It Works

```
 Define Products & Rates (Workspace Settings)
        │
        ▼
 Chat with AI → "Create a proposal for Client X, they need Package A + B"
        │
        ▼
 AI generates structured proposal with real pricing
        │
        ▼
 Edit in rich text editor → adjust prices, add notes
        │
        ▼
 Export to branded PDF
```

## Quick Start (Docker)

```bash
git clone https://github.com/khaledbashir/ownllm1.git
cd ownllm1

# Build the image
docker build -t ownllm .

# Create storage directory
mkdir -p storage && chown -R 1000:1000 storage

# Run
docker run -d \
  --name ownllm \
  -p 3001:3001 \
  -v $(pwd)/storage:/app/server/storage \
  -e STORAGE_DIR=/app/server/storage \
  ownllm
```

Visit `http://localhost:3001` to get started.

### With PostgreSQL (Production)

```bash
docker run -d \
  --name ownllm \
  -p 3001:3001 \
  -v $(pwd)/storage:/app/server/storage \
  -e STORAGE_DIR=/app/server/storage \
  -e DATABASE_URL="postgresql://user:password@host:5432/ownllm" \
  ownllm
```

## Environment Variables

Copy and configure your `.env` from the example:

```bash
cp docker/.env.example docker/.env
```

Key variables:

| Variable | Description | Default |
|----------|-------------|---------|
| `SERVER_PORT` | Server port | `3001` |
| `STORAGE_DIR` | Persistent storage path | `/app/server/storage` |
| `DATABASE_URL` | PostgreSQL connection string (optional, uses SQLite by default) | — |
| `LLM_PROVIDER` | LLM provider (`openai`, `anthropic`, `gemini`, `ollama`, etc.) | — |
| `OPEN_AI_KEY` | OpenAI API key (if using OpenAI) | — |
| `ANTHROPIC_API_KEY` | Anthropic API key (if using Claude) | — |
| `SIG_KEY` | Encryption key (auto-generated if not set) | — |
| `SIG_SALT` | Encryption salt (auto-generated if not set) | — |
| `JWT_SECRET` | JWT signing secret (needed if auth is enabled) | — |
| `DISABLE_TELEMETRY` | Set to `true` to disable anonymous telemetry | `false` |

See `docker/.env.example` for the full list of supported LLM/vector DB providers and their configuration.

## Development Setup

```bash
yarn setup          # Install deps and create .env files
yarn dev:server     # Start backend (port 3001)
yarn dev:frontend   # Start frontend (port 3000)
yarn dev:collector  # Start document processor
```

## Tech Stack

- **Frontend:** React + Vite + Tailwind CSS
- **Backend:** Node.js + Express
- **Database:** SQLite (default) or PostgreSQL via Prisma ORM
- **Rich Editor:** Yoopta / BlockNote
- **PDF Generation:** Puppeteer
- **Real-time:** WebSockets
- **Container:** Docker (Ubuntu Noble base)

## Project Structure

```
├── frontend/          # React + Vite UI
├── server/            # Node.js Express API
├── collector/         # Document processing service
├── docker/            # Docker build files and .env.example
├── embed/             # Embeddable chat widget
├── browser-extension/ # Chrome extension
└── pdf_export_api/    # PDF export service
```
